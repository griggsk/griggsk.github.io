    OSP Project Overview

    The Oregon Screening Project (OSP) goal is to enhance early developmental screening participation in Oregon. By offering an evidence-based resource library, age-appropriate activities, and two screening tools, OSP empowers parents and caregivers to understand their child's developmental progress, identify potential needs, and receive tailored recommendations. Additionally, OSP's administrative tools enable caregivers and researchers to monitor statewide screening progress.

    Faced with a decade's worth of technical debt and a critical server migration, the Oregon Screening Project (OSP) required more than an update; its architecture needed a complete redesign. The limitations of the existing legacy LAMP stack, which struggled to support both content and screening/research functionalities, necessitated a strategic separation of concerns. The solution involved migrating the content to Drupal CMS and rebuilding the core applications using Laravel and the TALL stack.

    This strategic architectural shift positions OSP for long-term sustainability and enhanced functionality. By leveraging modern technologies, OSP is now better equipped to serve parents, caregivers, and researchers across Oregon.

    Deliverables

    Key Deliverables

    My contributions spanned the entire project lifecycle, encompassing conception, planning, and implementation

    UX Research

    User and stakeholder research, including surveys, analytics, and interviews, defined OSP's audience and content needs. This informed the development of content models, user flows, and wireframes. User testing and card sorting then refined the information architecture, resulting in detailed mockups.

    UI Design

    Based on the established information architecture, the UI design focused on creating an intuitive and accessible user experience. This involved developing a comprehensive design system, including style guides and component libraries, and producing high-fidelity mockups that were refined through iterative reviews.

    Project Management

    Using an agile approach, iterative development and continuous collaboration, requirements were gathered and managed within a collaborative project management tool. Comprehensive documentation was maintained throughout the project lifecycle, including technical specifications, API documentation, and user guides, ensuring knowledge transfer and project sustainability.

    Web Development

    The UI/UX designs were implemented as a fully functional web application, featuring Laravel-built screening and research tools and Drupal CMS content delivery. Comprehensive testing validated functionality and performance across devices and browsers, including dedicated accessibility testing for WCAG compliance. Post-launch, ongoing support and maintenance ensured platform stability and addressed user needs.
