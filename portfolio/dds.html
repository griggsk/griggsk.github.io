    DDS Project Overview

    Educators in over 30,000 schools rely on the DIBELS® Data System (DDS) for critical data management and reporting of DIBELS® literacy assessments. For over a decade, in my roles as UI designer, software developer, and content coordinator, I spearheaded numerous high-impact customer-facing enhancements, feature implementations, and bug resolutions. Additionally, I led the content dissemination team, overseeing the management of content and training materials within the DDS platform.

    The DDS migration to an updated technology stack, aimed at resolving technical debt and database performance issues, involved migrating core functionality to the Zend Framework and refactoring the database. My contributions to the agile web development team encompassed code reviews, daily standups, and monthly sprint cycles, as well as technical documentation and code upkeep.

    As lead designer, I spearheaded a comprehensive redesign of the DDS user interface, focusing on significantly enhancing the user experience, accessibility, and mobile responsiveness. The redesign prioritized WCAG compliance, ensuring accessibility for all users, and implemented a responsive design approach to accommodate the diverse devices used by educators.

    In my role as accessibility coordinator, I proactively conducted accessibility testing, managed compliance documentation, and developed actionable remediation plans. I also took the initiative to create developer training materials and departmental accessibility policies, and provided critical contributions to RFPs and VPAT requests, driving a culture of accessibility throughout the organization.

    In 2022, the University of Oregon sold the DIBELS® Data System (DDS) to a private company. The DDS continues to operate on the UO platform, offering schools a low-cost option, while a partnership with the University's research arm ensures the development of new assessments and materials.

    Key Deliverables

    My contributions spanned the entire project lifecycle, encompassing conception, planning, and implementation.

    UX Research

    DDS's audience and content needs were determined through a user-driven research approach, incorporating surveys, analytics, and customer feedback. This research served as the foundation for developing content models, user flows, and wireframes. Through user testing and card sorting, the site's architecture was refined, which led to the creation of mockups and a content inventory.

    UI Design

    Utilizing finding from the UX research, the UI design was optimized for an intuitive and accessible experience, featuring streamlined navigation, efficient data entry, and accessible visualizations. To ensure consistent and rapid feature development, a robust style guide and pattern library were implemented. The redesign prioritized WCAG compliance and responsive design, guaranteeing accessibility across devices.

    Web Development

    My contributions to the development team included code migration, database design, and API development, followed by leading the implementation of numerous high-impact customer-facing enhancements, features, and bug fixes. I ensured efficient development through agile practices like code reviews, daily standups, and sprints, while also managing documentation and code upkeep. Utilizing standard DevOps tools, I streamlined deployments on an Apache server with CI/CD pipelines.

    Content Coordination

    I strategically managed DDS content and training materials, while driving impactful marketing campaigns and engaging webinars. By fostering close collaboration with the development team, I ensured feature updates and customer feedback were seamlessly integrated, resulting in a cohesive and responsive platform that enhanced user engagement.
